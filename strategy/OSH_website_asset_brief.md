# OSH Website V1 Asset Brief

This document tracks the numbered placeholders used by the first OSH website.
The public landing page intentionally uses placeholders until final photo, video,
render, and brand assets are ready.

| ID | Site Slot | Required Final Asset |
|---|---|---|
| LOGO-01 | Header/brand | Future OSH logo/mark; no current fire icon in V1. |
| PH-01 | Hero | Clean product-on-wall scene, OSH thermostat first-viewport signal. |
| PH-02 | Problem | Hallway-to-bathroom scenario visual: thermostat outside, sensor inside. |
| PH-03 | How it works | Simple 3-step visual: pair sensor, choose room, set temperature. |
| PH-04 | Product close-up | Device UI showing current temp, target temp, reference sensor state. |
| PH-05 | App dashboard | Phone mockup using current app dashboard screenshot as candidate source. |
| PH-06 | Temperature history | Phone mockup or video showing graph/history screen. |
| PH-07 | Wall UI video | Short muted loop of changing mode/schedule on AMOLED device UI. |
| PH-08 | Starter Kit | Kit contents: thermostat, floor NTC, wireless sensor, documentation. |
| PH-09 | Offline behavior | Diagram: internet unavailable, local thermostat/sensor/schedule still work. |
| PH-10 | Safety/trust | Real testing photo: load test, thermal check, bench setup, or lab scene. |
| PH-11 | Installation | Frame compatibility and mounting/depth visual. |
| PH-12 | Future marketplace | Placeholder for Prom/Rozetka links once product is actually listed. |

## Candidate Source Materials

- Device photo drafts: `design-sources/device/`
- Mobile app screenshot drafts: `design-sources/oshmobile_screenshots/`
- Current icon experiments: `design-sources/icon.png`, `design-sources/fire.png`

## Notes

- Keep the landing page free of direct commerce CTAs until marketplace pages are real.
- `design-sources/` contains drafts and source materials. Website code must not import files from this directory.
- Approved production images belong in `src/assets/site/`; production videos and downloadable files belong in `public/media/`.
- Register approved website media in the typed asset registry before replacing a placeholder.
- Do not overwrite existing source photos; create versioned filenames for edited assets.
